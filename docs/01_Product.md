## 목적

goodmoneying은 코인과 주식 투자 자동화를 위한 개인용 투자 데이터 플랫폼이다. 대상 종목과 코인을 발굴하고, 장기간의 과거 데이터(Back Data)를 기반으로 패턴을 분석하며, 매매전략을 개발하고, 개발된 전략을 봇(Bot)으로 시뮬레이션(Simulation)한 뒤 모의매매(Paper Trading)와 실제 자동매매(Live Trading)까지 단계적으로 확장하는 것을 목표로 한다.

초기 제품은 실제 주문 실행보다 먼저 신뢰 가능한 시장 데이터 기반과 운영 화면을 구축한다. 최종 제품은 업비트(Upbit) 코인, 국내 주식, 미국 주식, 가격/거래량, 호가(Orderbook), 수급, 공매도(Short Selling), 재무지표, 리포트, 뉴스, 공시, 대규모 언어 모델(LLM, Large Language Model) 분석 신호(Signal), 전략, 봇, 모의매매, 실거래를 모두 포함한다.

## 제품 범위

### 최종 제품 범위

- 업비트(Upbit) KRW 마켓 코인, 국내 주식, 미국 주식을 투자 분석 대상으로 지원한다.
- 가격, 거래량, 거래대금, 시가총액(Market Cap), 수급, 공매도(Short Selling), 재무지표를 수집한다.
- 코인은 24시간 365일 지속 수집한다.
- 국내 주식과 미국 주식은 시스템이 24시간 365일 실행되더라도 각 거래소의 정규장, 지원 가능한 확장장, 휴장일, 데이터 공급원 제약에 맞춰 수집 가능 시간에만 지속 수집한다.
- 호가(Orderbook)는 전략 입력으로 사용할 수 있는 요약 데이터부터 저장하고, 후속 단계에서 원천 스냅샷(Snapshot) 저장과 재생(Replay) 기반 분석으로 확장한다.
- 증권사 리포트, 뉴스, 공시를 주기적으로 수집하고 원천 데이터를 보존한다.
- 대규모 언어 모델(LLM, Large Language Model)은 리포트, 뉴스, 공시를 분석해 사람이 읽는 요약과 전략이 소비할 수 있는 구조화 신호(Signal)를 생성한다.
- LLM 신호는 주가 영향 카테고리(Category), 긍정/부정 방향, 강도(Strength), 신뢰도(Confidence), 기준 시점, 관련 종목을 포함한다.
- 가격 차트의 시계열(Time Series) 위에서 LLM 신호와 이후 가격/거래량 변화를 비교 분석할 수 있어야 한다.
- 사용자는 수집 데이터를 기반으로 시장 스캐닝(Scanning), 관심 대상 심층 분석, 전략 설계, 백테스트(Backtest)를 수행할 수 있어야 한다.
- 사용자는 봇(Bot)별 투자 가능 금액, 대상 시장, 대상 종목/코인, 리스크 한도, 실행 주기, 주문 방식, 전략 가중치(Weight)를 설정할 수 있어야 한다.
- 봇은 여러 전략을 파이프라인(Pipeline)으로 조합하고, 개별 전략 신호, 차단 조건, 리스크 조건, 가중치를 반영해 최종 매수/매도/보류 판단을 만들 수 있어야 한다.
- 봇 동작은 실제 주문 없이 시뮬레이션(Simulation)할 수 있어야 하며, 충분히 검증된 뒤 모의매매(Paper Trading)와 실제 자동매매(Live Trading)로 확장한다.

### 1차 MVP 범위

1차 최소 기능 제품(MVP, Minimum Viable Product)은 업비트 KRW 마켓 데이터 파이프라인과 운영 화면으로 제한한다. MVP는 M1부터 M3까지다.

- 업비트 KRW 마켓 24시간 거래대금 기준 상위 100개 후보를 표시한다.
- 시스템은 상위 100개 후보 중 기본 50개를 자동 체크한다.
- 사용자는 상위 100개 범위 안에서 체크를 변경해 최종 수집 대상 최대 50개를 조정할 수 있다.
- MVP 수집 데이터는 캔들(Candle), 현재가, 거래대금, 호가 요약(Orderbook Summary)으로 제한한다.
- 호가 요약은 최우선 매수/매도 가격과 수량, 스프레드(Spread), 상위 5호가 또는 10호가 누적 매수/매도 잔량, 호가 불균형(Imbalance), 갱신 시각, 수집 지연을 포함한다.
- 수집 데이터는 저장하고 조회할 수 있어야 한다.
- 데이터 품질(Data Quality)은 성공/실패, 지연, 결측, 중복, 최신성, 저장량을 포함한다.
- 첫 화면은 데이터 수집관리의 파이프라인 건강도 중심 운영 상태 대시보드다.
- MVP 화면은 운영 상태 대시보드, 수집 대상/설정 화면, 시장 리스트 화면, 코인 상세 화면을 포함한다.
- 제품 전체 좌측 내비게이션(Navigation)은 데이터 수집관리, 종목 발굴, 매매 전략, 봇 관리, 시스템 관리로 시작한다. MVP 기능은 데이터 수집관리 메뉴 안에서만 사용한다.

### MVP 제외 범위

MVP 제외 범위는 버리는 요구사항이 아니라 후속 마일스톤으로 이동하는 요구사항이다.

- 국내 주식과 미국 주식 수집
- 뉴스, 공시, 증권사 리포트 수집
- LLM 요약과 구조화 신호(Signal)
- 호가 원천 스냅샷(Snapshot) 전체의 고빈도 장기 보존
- 호가 원천 재생(Replay) 기반 정밀 백테스트(Backtest)
- 시장 스캐닝(Scanning), 전략 설계, 봇 설계, 모의매매, 실거래

### 계속 제외하는 범위

- 다중 사용자, 권한(Authorization), 과금(Billing), 팀 공유 기능은 개인 단일 사용자 제품 범위 밖으로 둔다.
- 이 제품은 투자 수익 보장이나 법적 의미의 매수/매도 추천을 목표로 하지 않는다.

## 사용자와 주요 시나리오

### 초기 사용자

초기 사용자는 본인 투자 분석과 자동화 기반 구축을 목표로 하는 개인 단일 사용자다.

### MVP 시나리오

1. **운영 상태 확인**
   - 사용자는 운영 상태 대시보드에서 업비트 수집 파이프라인이 정상인지 확인한다.
   - 시스템은 수집 성공/실패, 지연, 결측, 중복, 최신성, 저장량을 보여준다.
   - 현재가와 거래대금은 최근 1분에서 3분 안에 갱신되면 정상으로 본다.
   - 캔들은 선택한 단위의 마지막 완성 캔들이 저장되면 정상으로 본다.
   - 호가 요약은 최근 1분에서 3분 안에 갱신되면 정상으로 본다.

2. **수집 대상 조정**
   - 시스템은 업비트 KRW 마켓 24시간 거래대금 기준 상위 100개 후보를 보여준다.
   - 시스템은 기본 수집 대상 50개를 자동 체크한다.
   - 사용자는 상위 100개 범위 안에서 체크를 변경해 최종 수집 대상 최대 50개를 조정한다.
   - 수집 대상 설정은 운영 상태 대시보드와 시장 리스트가 공유하는 전역 수집 대상 풀(pool)이다.

3. **시장 리스트 탐색**
   - 사용자는 수집 대상 코인의 현재가, 거래대금, 등락률, 캔들/호가 최신성, 품질 상태를 표로 탐색한다.
   - 24시간 거래대금은 소수점 단위를 생략해 비교 가능한 표시값으로 본다.
   - 사용자는 품질 문제가 있는 코인을 빠르게 찾아 코인 상세 화면으로 이동할 수 있다.

4. **코인 상세 확인**
   - 사용자는 단일 코인의 캔들 차트, 현재가/거래대금 추이, 호가 요약, 수집 품질 이력을 확인한다.
   - 시장 리스트에서 코인을 누르면 코인 상세는 배경이 dimmed 처리된 레이어 팝업(layer popup)으로 표시된다.
   - 시스템은 호가 요약을 전략 입력으로 확장할 수 있는 형태로 보존한다.

5. **코인별 수집 계획 운영**
   - 사용자는 운영 상태 대시보드에서 수집 대상 코인별 행(row)을 펼쳐 수집 계획(Collection Plan)을 확인한다.
   - 코인별 수집 계획은 프리셋(Preset), 수집 시작 일시, 종료 일시 또는 현재(지속), 수집 방식, 상태를 포함한다.
   - 현재(지속) 수집은 KST(Korea Standard Time) 기준 전일 23:59:59까지의 구간을 진행 상태 기준으로 삼고, 저장과 내부 계산은 UTC(Coordinated Universal Time)를 사용한다.
   - 화면은 모든 시간 표시에 KST 또는 UTC 배지와 아이콘(icon)을 함께 보여준다.
   - 운영 상태 진행 상태는 단순한 진행률이 아니라 지정 기간 안에서 데이터가 있는 구간과 결측 구간을 구분한 구간형 진행 상태로 표시한다.

### 최종 제품 시나리오

1. **시장 스캐닝**
   - 사용자는 전체 시장에서 관심 후보를 찾기 위한 조건을 설정한다.
   - 시스템은 가격/거래량, 수급, 공매도, 재무지표, 호가 요약, LLM 신호를 기반으로 후보 종목과 코인을 추린다.

2. **관심 대상 심층 분석**
   - 사용자는 특정 국내 주식, 미국 주식, 코인을 선택한다.
   - 시스템은 장기 시계열(Time Series), 수급 변화, 공매도 변화, 호가 요약, 재무지표, 뉴스/공시/리포트 요약과 LLM 신호를 함께 보여준다.

3. **신호와 가격 영향 분석**
   - 사용자는 특정 카테고리(Category)의 긍정/부정 신호와 강도가 이후 가격, 수익률, 변동성, 거래량에 어떤 영향을 줬는지 확인한다.
   - 시스템은 신호 발생 시점을 가격 차트의 시계열(Time Series)에 정렬하고, 이후 구간의 가격/거래량 변화를 분석할 수 있어야 한다.

4. **전략 설계와 백테스트**
   - 사용자는 가격/거래량, 수급, 공매도, 재무지표, 호가 요약, LLM 신호를 조합해 전략 조건을 정의한다.
   - 시스템은 과거 데이터로 전략 성과를 검증하는 백테스트(Backtest)를 제공한다.
   - 호가 기반 전략은 호가 데이터 수집이 시작된 이후 기간에만 완전하게 검증할 수 있다는 제약을 명시한다.

5. **봇 설계**
   - 사용자는 봇(Bot)별 투자 가능 금액, 대상 시장, 대상 종목/코인, 리스크 한도, 실행 주기, 주문 방식, 전략 가중치(Weight)를 설정한다.
   - 같은 전략이라도 봇별로 파라미터(Parameter), 가중치, 투자 비중, 실행 방식을 다르게 적용할 수 있다.

6. **전략 파이프라인 구성**
   - 사용자는 여러 전략의 결과를 순차 또는 병렬 파이프라인(Pipeline)으로 조합한다.
   - 시스템은 개별 전략 신호, 차단 조건, 리스크 조건, 가중치를 반영해 최종 매수/매도/보류 판단을 만든다.

7. **봇 동작 시뮬레이션**
   - 사용자는 실제 주문 없이 봇이 시장 데이터나 과거 재생 데이터(Replay Data)를 입력받았을 때 어떻게 판단하고 행동하는지 시뮬레이션(Simulation)한다.
   - 시스템은 투자 가능 금액, 포지션(Position), 주문 후보, 체결 가정, 손익, 전략별 판단 근거, 최종 결정을 추적한다.
   - 시뮬레이션은 실제 주문 전 봇의 의사결정 흐름과 리스크를 검증하는 단계다.

8. **모의매매와 실거래**
   - 사용자는 검증된 봇을 모의매매(Paper Trading)로 실행한다.
   - 충분히 검증된 봇은 실제 자동매매(Live Trading)로 전환할 수 있다.

## 프론트엔드 화면 로드맵

이 문서는 프론트엔드(Frontend) 기술 스택을 결정하지 않고, 마일스톤별 화면과 사용자 경험(UX, User Experience) 요구사항만 정의한다. React, Next.js, Streamlit 같은 기술 선택은 아키텍처(Architecture) 문서 또는 ADR(Architecture Decision Record)에서 결정한다.

| 단계 | 화면 | 사용자가 확인하는 것 |
|---|---|---|

## 정책

- 시스템 프로세스는 24시간 365일 상시 실행된다.
- 코인 데이터는 업비트 시장 특성에 맞춰 24시간 365일 지속 수집한다.
- 국내 주식과 미국 주식은 거래소 장 운영시간, 휴장일, 데이터 공급원 제약에 맞춰 수집 가능 시간에만 지속 수집한다.
- 수집 가능 시간이 아닌 구간은 오류가 아니라 시장 휴장 또는 수집 대기 상태로 기록한다.
- 1차 MVP(Minimum Viable Product)는 업비트 KRW 마켓 데이터 파이프라인과 운영 화면으로 제한한다.
- 원천 데이터는 재처리와 감사(Audit)를 위해 가능한 한 보존한다.
- 호가(Orderbook) 데이터가 없는 과거 구간에서는 호가 기반 전략 검증을 완전하게 지원하지 않는다.
- LLM 분석 결과는 투자 판단의 보조 입력이며, 사실 데이터와 동일한 확정 정보로 취급하지 않는다.
- LLM 신호 카테고리(Category)는 버전 관리해야 하며, 카테고리 변경 시 과거 신호와 비교 가능성이 깨질 수 있음을 기록한다.
- 제품은 개인용 단일 사용자 플랫폼으로 시작하며, 다중 사용자 권한(Authorization), 과금(Billing), 팀 공유는 초기 범위에서 제외한다.
- 제품은 투자 수익을 보장하지 않으며, 매수/매도 추천의 법적 책임을 제품 목표로 두지 않는다.
