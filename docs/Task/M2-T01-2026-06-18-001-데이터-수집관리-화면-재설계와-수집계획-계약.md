## 목표

M2 진입 전에 MVP(Minimum Viable Product) 화면 구조를 좌측 내비게이션(Navigation) 기반으로 재정의하고, 데이터 수집관리 화면에서 코인별 수집 계획(Collection Plan), 구간형 진행 상태(Coverage Segment), 시장 리스트, 코인 상세 레이어 팝업(layer popup)을 실제 구현 가능한 계약과 화면으로 제공한다.

## 범위

- 포함: 좌측 제품 내비게이션 기본 메뉴(`데이터 수집관리`, `종목 발굴`, `매매 전략`, `봇 관리`, `시스템 관리`)
- 포함: MVP 기능은 `데이터 수집관리` 안에서만 사용
- 포함: 운영 상태 대시보드는 선택된 코인 최대 50개를 행(row)으로 보여주고, 각 행은 아코디언(Accordion)으로 상세를 펼침
- 포함: 코인별 수집 계획(Collection Plan)을 내부/외부 계약 모두 개별 코인 단위로 정의
- 포함: 수집 범위는 시작 일시부터 종료 일시 또는 현재(지속)까지 지원
- 포함: 현재(지속) 수집의 진행 기준은 화면 기준 KST(Korea Standard Time) 전일 23:59:59까지, 저장/계산 기준은 UTC(Coordinated Universal Time)
- 포함: KST/UTC 표시는 작은 배지 또는 아이콘으로 항상 구분
- 포함: 운영 상태 진행 상태는 단순 비율이 아니라 데이터가 있는 구간과 결측 구간을 구분한 구간형 진행 상태로 표시
- 포함: 수집 대상 설정은 상위 100개 후보 중 최대 50개 선택만 담당
- 포함: 시장 리스트는 수집된 데이터를 요약하고 24시간 거래대금은 소수점 없이 표시
- 포함: 시장 리스트에서 코인을 누르면 코인 상세를 dimmed 배경의 레이어 팝업으로 표시
- 포함: 프론트엔드(Frontend)는 통계 계산을 하지 않고 백엔드(Backend)가 제공한 화면용 View Model을 렌더링
- 포함: 백엔드는 조회 요청 시 큰 연산을 하지 않고, 수집/배치 시점에 계산해 저장할 수 있는 계약을 둠

## 현재 맥락

M1 구현은 탭(tab) 기반 운영 화면을 제공하지만, 제품 전체 구조의 좌측 내비게이션과 M2 이후 확장 메뉴가 없다. 기존 API는 `DashboardSummary.coverage.progressPercent` 중심이라 결측 구간을 시각화하는 데 필요한 `coverageSegments`와 코인별 수집 계획 View Model이 부족하다.

## 설계 메모

- 화면은 제품 전체 메뉴를 먼저 보여주되 MVP에서 활성화되는 영역은 `데이터 수집관리`로 제한한다.
- `수집 대상 설정`은 어떤 코인을 수집할지 정하는 전역 대상 풀(pool)이고, 코인별 수집 계획은 운영 상태 대시보드의 아코디언 상세에서 편집한다.
- 수집 계획은 코인별로 독립이다. `KRW-BTC`와 `KRW-ETH`는 서로 다른 시작 일시, 종료 일시, 현재(지속) 여부, 프리셋(Preset), 방식, 상태를 가질 수 있다.
- 프론트엔드는 백엔드가 제공한 `rangeLabel`, `progressLabel`, `coverageSegments.offsetPercent`, `coverageSegments.widthPercent`, `accTradePrice24hDisplay`를 그대로 표시한다.

## 계약 변경

- DB에 `collection_plans`, `collection_coverage_snapshots`, `collection_coverage_segments`를 추가한다.
- API `DashboardSummary`에 코인별 운영 행 `targets`를 추가한다.
- API `MarketListRow`에 소수점 없는 표시용 24시간 거래대금 `accTradePrice24hDisplay`를 추가한다.
- 활성 수집 대상 저장은 상위 100개 후보 중 최대 50개 선택으로 완화한다.

## 실패 케이스

- 수집 대상이 50개를 초과함
- 후보 유니버스 밖 코인을 수집 대상으로 저장함
- 코인별 수집 계획이 없는 활성 수집 대상이 운영 상태에 표시되지 않음
- 지속 수집 상태에서 KST/UTC 기준이 UI에 표시되지 않음
- 결측 구간이 `missing` 세그먼트로 제공되지 않아 화면이 빨간색 구간을 표시할 수 없음
- 프론트엔드가 API 값을 무시하고 직접 진행률 또는 24시간 거래대금을 계산함
- 코인 상세가 페이지 이동으로 열려 dimmed 레이어 요구를 충족하지 못함
