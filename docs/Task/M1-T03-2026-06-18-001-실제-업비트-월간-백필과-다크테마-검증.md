# 실제 업비트 월간 백필과 다크테마 검증 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** 실제 업비트(Upbit) API로 현재 시점 24시간 거래대금 상위 50개 코인의 2026년 1월 1분봉을 백필(Backfill)하고, 프론트엔드(Frontend)에서 다크테마(Dark Theme), 백필 진행률, 캔들 차트, 데이터 식별성을 검증한다.

**Architecture:** 운영 서버(Operations Server)는 백필 계획과 작업을 DB 계약(Contract)에 저장하고, 수집 워커(Collection Worker)는 DB에서 승인된 작업을 클레임(Claim)해 업비트 캔들 API를 `to` 기반 역방향 페이지네이션(Pagination)으로 호출한다. 초당 요청 제한(Rate Limit), 재시도(Retry), 부분 실패 기록, 대상별 진행률을 워커와 저장소(Repository) 사이의 작은 메서드로 나눈다.

**Tech Stack:** Python 3.14, FastAPI, PostgreSQL, httpx, React, React Query, Vitest, Playwright, Docker Compose.
