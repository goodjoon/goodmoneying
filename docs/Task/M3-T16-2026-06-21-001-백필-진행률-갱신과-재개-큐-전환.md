# M3-T16 백필 진행률 갱신과 재개/큐 전환

## 목표

Backfill 관리 화면에서 백필 작업(Backfill Job)의 진행률과 제어 상태가 실제 워커(Worker) 실행 상태를 따라 10초 주기로 갱신되게 하고, 멈춤(Pause)/중지(Stop) 후 다음 대기 작업이 즉시 실행 대상으로 전환되게 한다.

## 범위

- Backfill 관리 화면의 운영 스냅샷을 10초 주기로 자동 갱신
- `paused` 백필 작업에 재개(Resume) 버튼 표시 및 `resume` 제어 호출
- 축약된 대상 코인 목록에 전체 대상 코인 툴팁 표시
- 워커가 현재 작업의 멈춤/중지를 감지하면 같은 실행 틱에서 다음 `pending` 작업을 점유

## 비범위

- 백필 동시 실행 수 확대
- 백필 진행률 산식 변경
- DB 스키마(schema) 변경

## 현재 맥락

기존 Backfill 관리 화면은 대시보드 탭이 아닐 때 운영 스냅샷 자동 갱신이 꺼져 있어 진행률 변화가 화면에 반영되지 않는다. 또한 `paused` 상태에서 `resume` API는 존재하지만 화면 버튼이 없고, 워커는 제어된 작업 처리 후 다음 폴링까지 대기한다.

## 설계 메모

- `useOperationsConsole`의 자동 갱신 주기는 운영 상태 15초, Backfill 관리 10초로 분리한다.
- API 계약(Contract)의 `POST /v1/backfill/jobs/{jobId}/{action}`은 이미 `resume` 액션을 포함하므로 계약 변경은 없다.
- 화면의 전체 대상 코인 목록은 기본 브라우저 툴팁(title)으로 제공해 추가 API를 만들지 않는다.
- 워커는 멈춤/중지된 현재 작업을 더 처리하지 않고, 같은 `run_backfill_once` 호출 안에서 다음 대기 작업을 점유한다.

## 실패 케이스

- Backfill 관리 탭에서 실행 중 작업 진행률이 10초 후에도 다시 조회되지 않는다.
- `paused` 작업에 재개 버튼이 없어 사용자가 작업을 다시 실행할 수 없다.
- `외 n개`로 축약된 대상 코인의 전체 목록을 확인할 방법이 없다.
- 현재 작업을 멈춤/중지해도 다음 대기 작업이 다음 폴링 전까지 실행 대상으로 전환되지 않는다.

## 완료 기준

- Backfill 관리 화면에서 작업 목록과 진행률이 10초 주기로 다시 조회된다.
- `paused` 작업은 멈춤 버튼 대신 재개 버튼을 표시하고 `resume` 액션을 호출한다.
- 대상 코인 축약 텍스트에 전체 대상 코인 목록 툴팁이 표시된다.
- 멈춤/중지된 작업 다음의 `pending` 작업이 같은 워커 실행 틱에서 처리된다.
