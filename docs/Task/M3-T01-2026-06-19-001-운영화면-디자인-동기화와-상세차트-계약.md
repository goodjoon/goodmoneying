## 범위

- 운영 상태, 수집 대상/설정, 시장 리스트, 코인 상세 화면의 레이아웃과 시각 스타일을 디자인 이미지에 가깝게 변경한다.
- API 계약에 운영 헬스, 저장량, 실패율, 후보 품질, 커버리지, 상세 중복 행 같은 화면용 값을 추가한다.
- 티커 표시는 `KRW-BTC` 단일 문자열 대신 코인과 기준화폐를 분리해 표시한다.
- 시간 표시는 해당 시간 옆의 작은 KST 또는 UTC 라벨 하나만 보여준다.
- 코인 상세 캔들 차트는 TradingView Lightweight Charts 기반으로 렌더링하고 현재가 게이지와 볼륨 게이지를 함께 표시한다.

## 비범위

- 실제 주문, 전략, 봇, 국내/미국 주식 화면 구현
- 호가 원천 스냅샷 전체 장기 보존
- TradingView 호스팅 위젯 또는 MEXC 외부 데이터 직접 연결

## 현재 맥락

현재 API에는 `quoteCurrency`, `baseAsset`이 있으나 화면은 여러 곳에서 `marketCode`를 그대로 노출한다. 운영 화면은 디자인보다 단순한 아코디언/표 구조이며, 상세 차트는 직접 그린 SVG다. M2 아키텍처 기준은 화면용 View Model을 백엔드에서 제공하는 것이다.

## 설계 메모

- 저장 시각은 UTC를 유지하고, 업비트 KRW 화면 표시 기본값은 KST다.
- 화면에서 UTC/KST를 동시에 독립 배지로 노출하지 않고, 실제 시간 문자열 옆에 작은 라벨 하나만 붙인다.
- 차트는 백엔드 캔들 데이터를 사용하되 TradingView Lightweight Charts를 렌더링 엔진으로 사용한다.
- API 확장은 기존 필드를 제거하지 않는 호환 확장으로 처리한다.

## 계약 변경

- `DashboardTotals`에 정상/주의/장애 대상 수, 저장량, 실패율, 요청 수, rate limit 여유를 추가한다.
- `DashboardSummary`에 운영 헬스 카드 배열을 추가한다.
- `CandidateUniverseEntry`에 표시용 거래대금, 품질 상태, 수집 범위 표시값을 추가한다.
- `MarketListRow`에 커버리지와 저장량 표시값을 추가한다.
- `InstrumentDetailResponse`에 상세 화면 보조 지표를 추가한다.

## 실패 케이스

- 화면에 `KRW-BTC` 같은 market code가 주요 이름처럼 노출된다.
- 시간 영역에 KST와 UTC가 동시에 라벨처럼 노출된다.
- 디자인에 있는 숫자가 프론트엔드 임의 fixture가 아니라 API 응답으로 나오지 않는다.
- 캔들 차트가 비어 있거나 현재가 게이지가 표시되지 않는다.
