## 목표

개발자가 fixture 데이터와 실제 PostgreSQL 기반 개발 데이터를 혼동하지 않도록, 프로젝트 루트에서 infra와 app을 분리해 실행/중지/상태 확인할 수 있는 개발환경 제어 스크립트를 제공한다.

## 범위

- 포함: 프로젝트 루트 `dev.sh`
- 포함: 파라미터가 없을 때 사용법 출력
- 포함: `infra start|stop|restart|status`와 `postgres|all` 대상
- 포함: `app start|stop|restart|status`와 `api|web|worker|all` 대상
- 포함: 앱 로그 조회 명령
- 포함: 루트 `.env` 자동 로딩
- 포함: 기본값을 담은 `.env`와 커밋 가능한 `.env.sample`
- 포함: API가 기본적으로 PostgreSQL 개발 DB를 바라보도록 `GOODMONEYING_DATABASE_URL` 설정
- 포함: worker는 실제 업비트(Upbit) 수집 프로필(profile)을 기본값으로 사용하는 반복 실행 프로세스

## 비범위

- 운영 배포 스크립트
- PostgreSQL 볼륨 삭제
- Kubernetes 또는 원격 서버 배포

## 설계 메모

- infra는 Podman Compose로 관리한다. PostgreSQL은 앱 재시작과 무관하게 계속 켜둘 수 있어야 한다.
- app은 로컬 프로세스로 관리한다. API와 웹은 코드 수정 반영이 빠른 개발 서버를 사용한다.
- app PID와 로그는 `.dev/`에 저장하고 git에서는 제외한다.
- `.env`는 로컬 설정 파일이라 git에서 제외하고, 기본값 공유는 `.env.sample`로 한다.
- 셸에서 직접 지정한 환경변수는 `.env`보다 우선한다.
- `./dev.sh app start api`는 `GOODMONEYING_DATABASE_URL`을 설정해 fixture SQLite 저장소로 떨어지지 않게 한다.
- `./dev.sh app start worker`는 `GOODMONEYING_LIVE_UPBIT=1` 기본값으로 실제 업비트 API를 호출한다.

## 완료 기준

- `bash dev.sh`가 사용법을 출력한다.
- `bash dev.sh status`가 infra와 app의 상태를 출력한다.
- `bash dev.sh unknown`이 실패하고 사용법을 출력한다.
- 루트 `.env` 또는 `GOODMONEYING_ENV_FILE`로 지정한 파일의 환경변수가 적용된다.
- 셸 환경변수가 `.env` 값보다 우선한다.
- `bash -n dev.sh`가 통과한다.
- `uv run pytest tests/scripts/test_dev_script.py -q`가 통과한다.
