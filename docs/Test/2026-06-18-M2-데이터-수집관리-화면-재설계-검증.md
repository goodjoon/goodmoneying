## 검증 대상

- 데이터 수집관리 좌측 내비게이션(Navigation) 화면 구조
- 운영 상태 대시보드(Dashboard)의 코인별 아코디언(Accordion)
- 코인별 수집 계획(Collection Plan)과 구간형 진행 상태(Coverage Segment) API 응답
- 최대 50개 수집 대상 설정
- 시장 리스트(Market List)의 24시간 거래대금 표시값
- dimmed 배경의 코인 상세 레이어 팝업(Layer Popup)
- KST(Korea Standard Time)와 UTC(Coordinated Universal Time) 배지 표시
- 프론트엔드(Frontend)가 통계 계산을 하지 않고 백엔드(Backend) View Model을 렌더링하는 계약

## RED 확인

TDD(Test-Driven Development) 순서를 지키기 위해 구현 전에 다음 실패를 확인했다.

- `uv run pytest tests/contracts -q`: `DashboardSummary.targets`, `collection_plans`, `collection_coverage_segments`, 최대 50개 계약이 없어 실패
- `uv run pytest tests/api -q`: 대시보드 코인별 행과 50개 수집 대상 저장 계약이 없어 실패
- `uv run pytest tests/shared -q`: 저장소의 `collection_dashboard_targets()`와 `DashboardSummary.targets`가 없어 실패
- `npm test`: 좌측 내비게이션, 아코디언, KST/UTC 배지, 시장 리스트 모달 요구를 만족하지 못해 실패

## E2E 메모

초기 E2E(End-to-End)는 `playwright.config.ts`의 `reuseExistingServer`가 기존 5173/8000 포트 서버에 붙어 새 화면이 아닌 이전 화면을 검증하는 문제가 있었다. 테스트 신뢰성을 위해 Playwright 전용 포트 `15173` 웹 서버와 `18000` API 서버를 사용하도록 격리했다.

## 수동 검증

자동 E2E에서 다음 사용자 흐름을 확인했다.

- 데이터 수집관리 메뉴가 활성화되고 나머지 기본 메뉴는 비활성화됨
- 운영 상태 대시보드가 첫 화면으로 표시됨
- 코인별 수집 상태 행을 누르면 아코디언으로 수집 계획과 구간형 진행 상태가 펼쳐짐
- 수집 대상 설정에서 50개 선택 상태와 저장 동작이 유지됨
- 시장 리스트에서 코인을 누르면 dimmed 배경 위에 코인 상세 대화상자(Dialog)가 열림

## 미검증 항목

- 실제 장기 스케줄러(Scheduler)가 수집/배치 시점에 `collection_coverage_snapshots`, `collection_coverage_segments`를 갱신하는 흐름은 이번 Task 범위 밖이다.
- 실제 업비트 장기 백필(Backfill) 수행량과 속도 검증은 M1 백필 검증 문서에 따른다.
- `StarletteDeprecationWarning`은 FastAPI TestClient 의존성 경고이며 기능 실패는 아니다.
