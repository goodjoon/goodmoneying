## RED 확인

구현 전 `uv run pytest tests/scripts/test_dev_script.py -q`를 실행해 `dev.sh` 파일이 없어 3개 테스트가 실패하는 것을 확인했다.

## 수동 검증

`./dev.sh status` 출력에서 다음을 확인했다.

- `infra postgres` 상태가 표시됨
- `app api`, `app web`, `app worker` 상태가 표시됨
- 실행 중인 로컬 API와 웹 endpoint가 함께 표시됨
- 테스트 전용 `GOODMONEYING_ENV_FILE`을 통해 `.env` 값이 적용됨
- 셸 환경변수가 `.env` 값보다 우선함

## 미검증 항목

- 실제 `./dev.sh infra start`는 현재 Podman 컨테이너 상태를 바꿀 수 있어 이번 검증에서는 실행하지 않았다.
- 실제 `./dev.sh app start worker`는 업비트(Upbit) API를 호출할 수 있어 이번 검증에서는 실행하지 않았다.

개발환경 제어 스크립트의 사용법, 상태 조회, 오류 처리, 구문 검사는 통과했다.
