## 리뷰 대응

- 대시보드의 반복 데이터 타입 표시를 시장별 수집 상태로 바꿔 시장 코드, 최신성, 품질, 진행률을 바로 확인하게 했다.
- 시장 리스트는 활성 50개 전체를 표시하고, 행 클릭으로 해당 시장 상세 화면으로 이동하게 했다.
- 백필 job 제어 버튼은 상태별 가능 명령만 표시한다. `succeeded` job `#7`은 일시정지/중지 버튼을 표시하지 않는다.
- 상세 차트는 `2026년 1월 1분봉`, UTC 범위, 표시 표본 수와 저장 행 수를 명시한다.
- fresh DB에서 API fixture seed 이후 worker live refresh가 실행될 때 stale fixture 활성 대상이 남지 않도록, 최신 후보 유니버스 밖 활성 대상이 있으면 실제 상위 50개로 재동기화한다.
- `pause/stop`된 백필 job 상태가 target 완료 갱신으로 `running/succeeded`로 되살아나지 않게 저장소와 worker 루프를 수정했다.
- Postgres 백필 계획은 프로세스 메모리가 아니라 `backfill_jobs.status='planned'` 행으로 저장하고 승인 시 `pending`으로 전환한다.
- 업비트 rate limit은 `Remaining-Req`의 `sec=0`과 418 본문 차단 시간을 반영한다.
- API 오류 응답은 계약처럼 최상위 `code/message` 형태로 통일했다.

## 남은 개선 여지

- 캔들 차트는 월간 1분봉 전체를 96개 표본으로 표시하는 MVP 수준이다. 축, 툴팁, 줌, 기간 선택은 후속 단계에서 개선한다.

## 결론

실제 업비트 API로 현재 상위 50개 KRW 시장을 선정했고, 2026년 1월 1분봉 백필을 완료했다. PostgreSQL에는 50개 target의 작업 성공 상태와 40개 시장의 1,203,420개 캔들이 저장되었으며, API와 웹 E2E, 브라우저 화면 검증까지 통과했다.
