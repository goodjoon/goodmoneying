# 백필 진행률 갱신과 재개/큐 전환

## 변경 요약

- Backfill 관리 화면에서 운영 스냅샷을 10초 주기로 다시 조회해 백필 작업별 진행률이 갱신되게 했다.
- `paused` 백필 작업에는 멈춤(Pause) 버튼 대신 재개(Resume) 버튼을 표시하고 기존 `resume` 제어 API를 호출하게 했다.
- 대상 코인이 4개를 초과해 `외 n개`로 축약될 때 전체 대상 코인 목록을 `title` 툴팁으로 노출했다.
- 백필 워커(Backfill Worker)가 현재 작업의 멈춤/중지를 감지하면 같은 `run_backfill_once` 호출 안에서 다음 `pending` 작업을 점유하도록 했다.

## 계약/아키텍처 영향

- API 계약 변경 없음. `POST /v1/backfill/jobs/{jobId}/{action}`의 `resume` 액션은 기존 계약에 포함되어 있었다.
- DB 스키마(schema) 변경 없음.
- 백필 동시성 정책은 변경하지 않고, 단일 워커가 제어된 작업을 건너뛰어 다음 작업을 바로 처리하는 동작만 보강했다.

## 후속 리스크

- 현재 전체 대상 코인 목록은 기본 브라우저 툴팁(title)이다. 고정 위치, 긴 목록 스크롤, 모바일 터치 표시가 필요해지면 별도 tooltip 컴포넌트가 필요하다.
