## 변경 요약

- 프로젝트 루트에 `dev.sh`를 추가했다.
- infra 제어 명령을 `./dev.sh infra start|stop|restart|status [postgres|all]` 형태로 제공한다.
- app 제어 명령을 `./dev.sh app start|stop|restart|status [api|web|worker|all]` 형태로 제공한다.
- 파라미터가 없으면 사용법을 출력하도록 했다.
- 앱 실행 시 PID와 로그를 `.dev/` 아래에 저장한다.
- 루트 `.env` 파일이 있으면 자동으로 읽도록 했다.
- 기본 개발 설정을 `.env`와 `.env.sample`에 추가했다.
- `.env`는 git에서 제외하고 `.env.sample`만 공유 대상으로 둔다.
- 셸에서 지정한 환경변수가 `.env` 값보다 우선하도록 했다.
- API 로컬 실행은 기본적으로 PostgreSQL 개발 DB를 바라보도록 `GOODMONEYING_DATABASE_URL`을 설정한다.
- README에 `dev.sh` 중심 개발환경 실행 방법을 반영했다.

## 리스크

- `podman compose` 또는 `podman-compose`가 없는 환경에서는 infra 명령이 실패한다.
- worker는 기본적으로 `GOODMONEYING_LIVE_UPBIT=1`을 사용하므로 실제 외부 API 호출이 발생한다.
- 기존에 수동으로 띄운 API/웹 프로세스는 PID 파일이 없을 수 있어 상태/중지 명령이 포트 기반 보조 탐지를 사용한다.
- `.env`는 로컬 설정 파일이므로 실제 비밀값을 넣을 수 있지만, `.env.sample`에는 비밀값을 넣지 않는다.

## 후속 작업

- 장기적으로 `worker`를 증분 수집 루프와 백필 루프로 명확히 분리할지 결정한다.
- API 헬스 체크에 현재 저장소 종류(SQLite/PostgreSQL)와 데이터 소스(fixture/live)를 노출할지 검토한다.
