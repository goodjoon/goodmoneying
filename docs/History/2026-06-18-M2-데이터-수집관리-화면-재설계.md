## 변경 요약

- 제품 전체 좌측 내비게이션(Navigation)을 추가하고 MVP(Minimum Viable Product) 활성 메뉴를 `데이터 수집관리`로 제한했다.
- 데이터 수집관리 내부에 `운영 상태 대시보드`, `수집 대상 설정`, `시장 리스트` 화면을 구성했다.
- 운영 상태 대시보드는 선택된 최대 50개 코인을 코인별 행(row)으로 표시하고, 행을 누르면 아코디언(Accordion)으로 세부 상태를 펼친다.
- 코인별 수집 계획(Collection Plan), 데이터별 수집 상태, 결측 구간을 포함한 구간형 진행 상태(Coverage Segment)를 백엔드(Backend) View Model로 제공했다.
- 수집 범위의 KST(Korea Standard Time) 표시와 UTC(Coordinated Universal Time) 저장/계산 기준을 화면 배지로 구분했다.
- 시장 리스트는 백엔드가 제공하는 `accTradePrice24hDisplay`를 사용해 24시간 거래대금을 소수점 없이 표시한다.
- 시장 리스트의 코인 선택은 페이지 전환이 아니라 dimmed 배경의 레이어 팝업(Layer Popup)으로 코인 상세를 연다.
- Playwright E2E(End-to-End)는 기존 개발 서버 재사용 문제를 피하기 위해 전용 포트로 격리했다.

## 리스크

- 이번 구현은 수집/배치 시점의 계산 결과를 저장할 계약과 기본 View Model을 제공하지만, 실제 장기 배치 계산기는 후속 Task에서 구체화해야 한다.
- 수집 계획 수정 화면은 MVP 화면 구조와 로컬 입력 상태를 제공한다. 실제 계획 저장 API는 후속 Task에서 별도 계약으로 확장해야 한다.
- Playwright E2E는 테스트 전용 포트 `15173`/`18000`을 사용한다. 로컬에서 해당 포트를 다른 프로세스가 점유하면 테스트 전 환경 변수를 조정해야 한다.

## 후속 작업

- 코인별 수집 계획 저장 API와 배치 반영 흐름을 구현한다.
- `collection_coverage_snapshots`, `collection_coverage_segments`를 실제 수집/백필 완료 시 갱신하는 배치 작업을 구현한다.
- 백필 계획 생성 이후 승인/실행 UI를 아코디언 상세 안에서 이어서 조작할 수 있게 확장한다.
