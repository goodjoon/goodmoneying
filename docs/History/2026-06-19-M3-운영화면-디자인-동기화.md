## 변경 요약

- 운영 화면을 디자인 이미지 기준의 다크 운영 콘솔로 재구성했다.
- 대시보드(Dashboard), 수집 대상 설정, 시장 리스트(Market List), 상세 대화상자(Dialog)를 같은 시각 체계로 맞췄다.
- `KRW-BTC` 같은 마켓 코드(Market Code)를 주요 이름으로 노출하지 않고 `BTC / KRW` 형태로 코인과 기준화폐를 분리해 표시한다.
- 시간 표시는 KST(Korea Standard Time) 또는 UTC(Coordinated Universal Time) 중 하나의 작은 라벨만 시간 문자열 옆에 붙이도록 정리했다.
- 상세 화면은 TradingView Lightweight Charts 기반 캔들 차트(Candlestick Chart)와 현재가/거래량 게이지(Gauge)를 사용한다.

## 리스크

- 저장량과 실패율은 현재 저장소의 수집 스냅샷을 바탕으로 한 운영 View Model 값이다. 장기 운영 데이터가 충분히 쌓이면 수치 산식 조정이 필요할 수 있다.
- 외부 TradingView 호스팅 위젯 또는 MEXC 외부 데이터는 연결하지 않았다. 내부 API 캔들 데이터를 TradingView Lightweight Charts 렌더링 엔진으로 표시한다.
- Codex 명령 실행 환경은 `nohup`으로 시작한 자식 프로세스를 호출 종료 시 정리했다. 현재 개발 서버는 `launchctl submit` 라벨(`com.goodmoneying.dev.api`, `com.goodmoneying.dev.web`, `com.goodmoneying.dev.worker`)로 실행 중이다.

## 후속 작업

- 실제 운영 데이터 누적 후 저장량, 실패율, rate limit 여유 표시 산식의 운영 적합성을 재검토한다.
- 추가 거래소(Exchange) 또는 차트 공급자 연동이 필요해지면 ADR(Architecture Decision Record)로 외부 위젯과 내부 렌더링 방식의 장단점을 비교한다.
